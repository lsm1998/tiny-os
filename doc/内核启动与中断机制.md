## GDT表

GDT（Global Descriptor Table，全局描述符表）是 x86 保护模式下用于描述内存段的信息表。

它的作用是告诉 CPU：
1. 某一段内存是代码段还是数据段
2. 该段的起始地址和长度
3. 该段是否可访问、可读写
4. 该段的权限级别属于内核还是用户

在本项目中，GDT 主要用于建立内核代码段和内核数据段，为 CPU 提供基本的地址映射与权限控制。

## IDT表

IDT（Interrupt Descriptor Table，中断描述符表）是 x86 保护模式下用于描述中断和异常处理入口的信息表。

它的作用是告诉 CPU：
1. 某个中断号或异常号对应哪个处理函数
2. 进入中断后应切换到哪个代码段执行
3. 该中断门是否有效，以及允许的权限级别
4. 当异常或硬件中断发生时，控制权应如何转交给内核

在本项目中，IDT 主要用于注册 CPU 异常和外部中断处理函数，并配合 PIC 完成中断响应。
