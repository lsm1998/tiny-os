## 什么是QEMU

QEMU 是一个硬件模拟器 / 虚拟机工具，写一个操作系统面对的是裸机环境，QEMU 能提供一台安全、可调试、可重复启动的虚拟机器，我们通常需要先将代码编译并制作成可启动镜像文件，然后使用QEMU虚拟机启动

```bash
# 启动一个 x86_64 架构的虚拟机，并将 os.img 作为 raw 格式的虚拟磁盘挂载
qemu-system-x86_64 -drive format=raw,file=os.img
```

## dd命令概述

`dd` 是一个用于转换和复制文件的命令行工具，常用于创建磁盘镜像、备份数据、以及在操作系统开发中制作可启动镜像文件。它的基本语法如下：

```bash
dd if=<输入文件> of=<输出文件> [options]
```

- `if`：指定输入文件，可以是一个普通文件、设备文件或标准输入
- `of`：指定输出文件，可以是一个普通文件、设备文件或标准输出
- `bs`：设置块大小，默认是 512 字节，可以根据需要调整
- `count`：指定要复制的块数
- `seek`：在输出文件中跳过指定数量的块，常用于将数据写入特定位置
- `conv`：指定转换选项，如 `notrunc`（不截断输出文件）等

## elf文件概述

首先elf是一种常见的可执行文件格式，它是我们编译内核过程中的中间产物，包含了程序的代码、数据、符号表等信息，我们需要通过它进行debug调试

我们最终通过 `objcopy` 将其转换为.bin格式的裸二进制文件，然后才能被引导扇区加载并执行

## 如何制作可启动镜像文件

一个最简单的操作系统通常包含引导扇区和内核文件，当 QEMU 启动时，会把 os.img 当作一块硬盘，首先读取第一个扇区，也就是前 512 字节的内容。如果这个扇区中包含可执行的引导代码，并且最后两个字节是启动签名 0x55AA，那么 BIOS 就会认为这是一块可启动磁盘，并开始执行其中的引导程序

假定我们有一个简单的引导扇区代码 `boot.asm` 和一个内核入口汇编文件 `start.asm`，以及一个内核 C 文件 `kernel.c`，我们可以使用以下命令进行编译：
```bash
# 1. 编译 bootloader
nasm -f bin boot.asm -o boot.bin

# 2. 编译内核入口汇编
nasm -f elf32 start.asm -o start.o

# 3. 编译内核 C 文件
x86_64-elf-gcc -c kernel.c -o kernel.o -ffreestanding -m32

# 4. 链接生成 ELF 内核
x86_64-elf-ld -m elf_i386 -Ttext 0x10000 start.o kernel.o -o kernel.elf

# 5. 转换为裸二进制内核
x86_64-elf-objcopy -O binary kernel.elf kernel.bin
```

开始制作可启动镜像文件
```bash
# 1. 创建一个空白镜像文件，大小为 16MB
dd if=/dev/zero of=os.img bs=1M count=16

# 2. 将 bootloader 写入第一个扇区
dd if=boot.bin of=os.img conv=notrunc

# 3. 将 kernel 写入后续扇区
dd if=kernel.bin of=os.img seek=1 conv=notrunc
```
